Type: MsgLinuxSocketUsage

Summaries the socket usage across the system.

OffsetSizeIDTypeNote
0 ... avg_queue_depth u4le average socket queue depths across all sockets on the system
4 ... max_queue_depth u4le the max queue depth seen within the reporting period
8 ... socket_state_counts u2le A count for each socket type reported in the `socket_types_reported` field, the first entry corresponds to the first enabled bit in `types_reported`.
40 ... socket_type_counts u2le A count for each socket type reported in the `socket_types_reported` field, the first entry corresponds to the first enabled bit in `types_reported`.

Type: MsgLinuxSysState

This presents a summary of CPU and memory utilization, including a timestamp.

OffsetSizeIDTypeNote
0 ... mem_total u2le total system memory, in MiB
2 ... pcpu u1 percent of CPU used, expressed as a fraction of 256
3 ... pmem u1 percent of memory used, expressed as a fraction of 256
4 ... procs_starting u2le number of processes that started during collection phase
6 ... procs_stopping u2le number of processes that stopped during collection phase
8 ... pid_count u2le the count of processes on the system
10 ... time u4le timestamp of message, refer to flags field for how to interpret
14 ... flags u1 flags

Type: MsgLinuxProcessSocketQueues

Top 10 list of sockets with deep queues.

OffsetSizeIDTypeNote
0 ... index u1 sequence of this status message, values from 0-9
1 ... pid u2le the PID of the process in question
3 ... recv_queued u2le the total amount of receive data queued for this process
5 ... send_queued u2le the total amount of send data queued for this process
7 ... socket_types u2le A bitfield indicating the socket types used: 0x1 (tcp), 0x2 (udp), 0x4 (unix stream), 0x8 (unix dgram), 0x10 (netlink), and 0x8000 (unknown)
9 ... socket_states u2le A bitfield indicating the socket states: 0x1 (established), 0x2 (syn-sent), 0x4 (syn-recv), 0x8 (fin-wait-1), 0x10 (fin-wait-2), 0x20 (time-wait), 0x40 (closed), 0x80 (close-wait), 0x100 (last- ack), 0x200 (listen), 0x400 (closing), 0x800 (unconnected), and 0x8000 (unknown)
11 ... address_of_largest str(ascii) Address of the largest queue, remote or local depending on the directionality of the connection.
??? ... cmdline str(ascii) the command line of the process in question

Type: MsgLinuxProcessFdSummary

Summary of open file descriptors on the system.

OffsetSizeIDTypeNote
0 ... sys_fd_count u4le count of total FDs open on the system
4 ... most_opened str(ascii) A null delimited list of strings which alternates between a string representation of the process count and the file name whose count it being reported. That is, in C string syntax "32\0/var/log/syslog\012\0/tmp/foo\0" with the end of the list being 2 NULL terminators in a row.
