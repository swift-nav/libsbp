Navigation format specification

Type: Navigation

OffsetSizeIDTypeNote

Type: MsgPosLlhDepA

This position solution message reports the absolute geodetic coordinates and the status (single point vs pseudo-absolute RTK) of the position solution. If the rover receiver knows the surveyed position of the base station and has an RTK solution, this reports a pseudo-absolute position solution using the base station position and the rover's RTK baseline vector. The full GPS time is given by the preceding MSG_GPS_TIME with the matching time-of-week (tow).

OffsetSizeIDTypeNote
0 ... tow u4le GPS Time of Week
4 ... lat f8le Latitude
12 ... lon f8le Longitude
20 ... height f8le Height
28 ... h_accuracy u2le Horizontal position accuracy estimate (not implemented). Defaults to 0.
30 ... v_accuracy u2le Vertical position accuracy estimate (not implemented). Defaults to 0.
32 ... n_sats u1 Number of satellites used in solution.
33 ... flags u1 Status flags

Type: MsgProtectionLevelDepA

This message reports the local vertical and horizontal protection levels associated with a given LLH position solution. The full GPS time is given by the preceding MSG_GPS_TIME with the matching time-of-week (tow).

OffsetSizeIDTypeNote

6 ... hpl u2le Horizontal protection level
8 ... lat f8le Latitude

Type: MsgPosLlhCov

This position solution message reports the absolute geodetic coordinates and the status (single point vs pseudo-absolute RTK) of the position solution as well as the upper triangle of the 3x3 covariance matrix. The position information and Fix Mode flags follow the MSG_POS_LLH message. Since the covariance matrix is computed in the local-level North, East, Down frame, the covariance terms follow that convention. Thus, covariances are reported against the "downward" measurement and care should be taken with the sign convention.

OffsetSizeIDTypeNote
0 ... tow u4le GPS Time of Week
4 ... lat f8le Latitude
12 ... lon f8le Longitude
20 ... height f8le Height above WGS84 ellipsoid
28 ... cov_n_n f4le Estimated variance of northing
32 ... cov_n_e f4le Covariance of northing and easting
36 ... cov_n_d f4le Covariance of northing and downward measurement
40 ... cov_e_e f4le Estimated variance of easting
44 ... cov_e_d f4le Covariance of easting and downward measurement
48 ... cov_d_d f4le Estimated variance of downward measurement
52 ... n_sats u1 Number of satellites used in solution.
53 ... flags u1 Status flags

Type: MsgPoseRelative

This solution message reports the relative pose of a sensor between two time instances. The relative pose comprises of a rotation and a translation which relates the sensor (e.g. camera) frame at a given time (first keyframe) to the sensor frame at another time (second keyframe). The relative translations is a 3x1 vector described in the first keyframe. Relative rotation is described by a quaternion from second keyframe to the first keyframe.

OffsetSizeIDTypeNote
0 ... tow u4le GPS Time of Week
4 ... sensor_id u1 ID of the sensor producing this message
5 ... timestamp_1 u4le Timestamp of first keyframe
9 ... timestamp_2 u4le Timestamp of second keyframe
13 ... trans s4le Relative translation [x,y,z] described in first keyframe
25 ... w s4le Real component of quaternion to describe relative rotation (second to first keyframe)
29 ... x s4le 1st imaginary component of quaternion to describe relative rotation (second to first keyframe)
33 ... y s4le 2nd imaginary component of quaternion to describe relative rotation (second to first keyframe)
37 ... z s4le 3rd imaginary component of quaternion to describe relative rotation (second to first keyframe)
41 ... cov_r_x_x f4le Estimated variance of x (relative translation)
45 ... cov_r_x_y f4le Covariance of x and y (relative translation)
49 ... cov_r_x_z f4le Covariance of x and z (relative translation)
53 ... cov_r_y_y f4le Estimated variance of y (relative translation)
57 ... cov_r_y_z f4le Covariance of y and z (relative translation)
61 ... cov_r_z_z f4le Estimated variance of z (relative translation)
65 ... cov_c_x_x f4le Estimated variance of x (relative rotation)
69 ... cov_c_x_y f4le Covariance of x and y (relative rotation)

Type: MsgVelBody

This message reports the velocity in the Vehicle Body Frame. By convention, the x-axis should point out the nose of the vehicle and represent the forward direction, while as the y-axis should point out the right hand side of the vehicle. Since this is a right handed system, z should point out the bottom of the vehicle. The orientation and origin of the Vehicle Body Frame are specified via the device settings. The full GPS time is given by the preceding MSG_GPS_TIME with the matching time-of-week (tow). This message is only produced by inertial versions of Swift products and is not available from Piksi Multi or Duro.

OffsetSizeIDTypeNote
0 ... tow u4le GPS Time of Week
4 ... x s4le Velocity in x direction
8 ... y s4le Velocity in y direction
12 ... z s4le Velocity in z direction
16 ... cov_x_x f4le Estimated variance of x
20 ... cov_x_y f4le Covariance of x and y
24 ... cov_x_z f4le Covariance of x and z
28 ... cov_y_y f4le Estimated variance of y
32 ... cov_y_z f4le Covariance of y and z
36 ... cov_z_z f4le Estimated variance of z
40 ... n_sats u1 Number of satellites used in solution
41 ... flags u1 Status flags

Type: MsgGpsTimeDepA

This message reports the GPS time, representing the time since the GPS epoch began on midnight January 6, 1980 UTC. GPS time counts the weeks and seconds of the week. The weeks begin at the Saturday/Sunday transition. GPS week 0 began at the beginning of the GPS time scale. Within each week number, the GPS time of the week is between between 0 and 604800 seconds (=60*60*24*7). Note that GPS time does not accumulate leap seconds, and as of now, has a small offset from UTC. In a message stream, this message precedes a set of other navigation messages referenced to the same time (but lacking the ns field) and indicates a more precise time of these messages.

OffsetSizeIDTypeNote
0 ... wn u2le GPS week number
2 ... tow u4le GPS time of week rounded to the nearest millisecond
6 ... ns_residual s4le Nanosecond residual of millisecond-rounded TOW (ranges from -500000 to 500000)
10 ... flags u1 Status flags (reserved)

Type: MsgVelCog

This message reports the receiver course over ground (COG) and speed over ground (SOG) based on the horizontal (N-E) components of the NED velocity vector. It also includes the vertical velocity coordinate. A flag is provided to indicate whether the COG value has been frozen. When the flag is set to true, the COG field is set to its last valid value until the system exceeds a minimum velocity threshold. No other fields are affected by this flag. The NED coordinate system is defined as the local WGS84 tangent plane centered at the current position. The full GPS time is given by the preceding MSG_GPS_TIME with the matching time- of-week (tow). Note: course over ground represents the receiver's direction of travel, but not necessarily the device heading.

OffsetSizeIDTypeNote
0 ... tow u4le GPS Time of Week
4 ... cog u4le Course over ground relative to north direction
8 ... sog u4le Speed over ground (based on horizontal velocity)
12 ... v_up s4le Vertical velocity component (positive up)
16 ... cog_accuracy u4le Course over ground estimated standard deviation
20 ... sog_accuracy u4le Speed over ground estimated standard deviation
24 ... v_up_accuracy u4le Vertical velocity estimated standard deviation
28 ... flags u2le Status flags

Type: MsgGpsTimeGnss

This message reports the GPS time, representing the time since the GPS epoch began on midnight January 6, 1980 UTC. GPS time counts the weeks and seconds of the week. The weeks begin at the Saturday/Sunday transition. GPS week 0 began at the beginning of the GPS time scale. Within each week number, the GPS time of the week is between between 0 and 604800 seconds (=60*60*24*7). Note that GPS time does not accumulate leap seconds, and as of now, has a small offset from UTC. In a message stream, this message precedes a set of other navigation messages referenced to the same time (but lacking the ns field) and indicates a more precise time of these messages.
